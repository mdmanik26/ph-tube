The scope of var, let, and const :

Before es6 came out, var was used everywhere. With the advent of es6, let and const have largely eliminated the problems var had. But that doesn't mean that the use of var has completely disappeared. Some places require var in field specials.

The scope of var, let and const:
The scope of a variable means in which part of the code the variable can be used. The scope of var is 1. global and 2. inside the function.
If var is declared inside a function then its scope is limited only inside that function. But if it is declared outside a function, it can be used anywhere in the code. var can be both re-declared and updated. var is hosting, which keeps code's variable and function declarations above their scope.

On the other hand let and const are block scope. They are limited to the block in which they are declared. Variables declared with let can be updated within that block but not re-declared. But it can be declared inside any other block. Const cannot be updated or re-declared within its own block. But it can be declared in any other block.

That is why var cannot be excluded due to some problems. We can use var when there may be variables that need to be updated or re-declared depending on the context.

The use cases of null and undefined :

Null means nothing in JavaScript. It refers to something that no longer exists. The value of any variable can be null. This is done when the value of a variable is not useful now but may be useful later. When we need the value of that variable we can use it by setting something else instead of null. Null is treated as an object in JavaScript.

On the other hand, undefined in JavaScript is a variable that has no value. By setting the value of a variable to undefined, the value of that variable can be cleared. In other words, undefined means that which has not yet been determined. That is, by setting undefined as the value of a variable, that variable value is kept blank and then the value of that variable is updated. Also null and undefined can be used in different ways in the field.

REST API :

The full form of API is Application Programming Interface. We can call API as a medium or method or system. That is, the medium through which we can collect data from any platform and use it on our own platform is called API.

The full form of REST API is Representational State Transfer Application Programming Interface. Rest APIs and APIs work similarly to carriers. From time to time we share our information on different platforms for our work. For example, Google account. After we provide information to Google Account, Google Account stores that information. Again, if I want to do any work on a new platform, the new platform may ask for our information. In that case, we can tell the Google account to give our information to the new platform without giving the information ourselves. In this case, the medium through which information is exchanged between the Google account and the new platform is called Rest API.
